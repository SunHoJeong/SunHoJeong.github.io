About B.G.M.

Battle Grounds Moim

Battle Grounds APPLICATION

B.G.M.은 배틀그라운드 유저들을 위한 어플리케이션입니다.
당신이 최후의 1인이 될 수 있도록 전적검색, 같이하기 게시판, 지도 게시판, 영상 게시판을 제공합니다.
배틀그라운드 유저들에게 다양한 정보와 영상, 소통할 수 있는 공간을 제공함을 목표로 합니다

당신의 승리에 조금이라도 기여할 수 있는 B.G.M.이 되겠습니다

사용기술

PUBG TRACKER API

배틀그라운드 유저의 전적 검색을 하기 위해서 API를 사용하였습니다. 사용자의 게임 데이터를 JSON 형식으로 넘겨줍니다.
게임 데이터를 배틀그라운드 제작사인 '블루홀'에서 직접 제공하는것이 아니여서 가끔 서버가 터지거나 느린 경우가 많았습니다.

Retrofit & GsonConverter

Server와 통신하고 Json data를 Java Object로 변환하기 위해 Retrofit & GsonConverter를 사용하였습니다.
배틀그라운드 유저의 게임 데이터가 워낙 방대하기 때문에 이를 받아오고, 가공하는 과정에서 많은 시간을 들였습니다.

Firebase Realtime-Database, Cloud Messaging

팀원을 구하고자하는 유저들에게 실시간 데이터베이스를 이용해 실시간 게시판을 제공해주고 싶었습니다.
또한 손쉬운 글 작성과 댓글로의 참여, 이러한 이벤트들을 사용자에게 알려주는 FCM push알람으로
유저들이 간편하고 빠르게 원하는 팀원을 찾을 수 있도록 도와주고 싶었습니다.

YOUTUBE API & DRAGGABLE PANEL

유저들이 게임 중간 중간에 생기는 짜투리 시간을 의미있게 보낼 수 있도록 하고 싶었습니다.
짧은 하이라이트 영상들로 구성해 지루하지 않도록 하고 싶었습니다.
유투브 API를 사용한 만큼 UI도 유투브와 유사하게 꾸미고자 Draggable Panel을 사용하게 되었습니다.

STETHO & Firebase crach

저는 API를 사용하기 위해 API key가 필요했습니다. key가 필요한 API들을 이용하는 과정에 있어서
서버와 안드로이드 간의 왕래하는 데이터를 확인하는것이 쉽지만은 않았습니다. 그래서 STETHO를 사용하게 되었습니다.
안드로이드에서 서버로 어떤 데이터가 전송되었고, 서버에서 안드로이드로 어떤 데이터가 전송되는지,
어떤 http 오류가 발생했는지와 같은 정보들을 명확하게 알 수 있었고 서버와 안드로이드 어느쪽에 문제가 발생했는지
알아차리기가 쉬웠습니다.

Parceler & Event Bus

방대한 유저 전적 데이터를 받아온 후에 사용자에게 보여주기 위해서 데이터를 뿌려주는 과정이 필요했습니다.
이러한 과정에서 액티비티간 오브젝트 전달이 필요하였고 이를 위해서 Parceler를 사용하였습니다.
또한 계층적으로 연관되어있지 않은 구조에서 이벤트를 전달해야하는 경우들이 생겼습니다.
이러한 문제를 해결하고자 Event Bus를 사용했습니다. 해당 이벤트에 대한 구독자를 지정하고 해당 이벤트가 발생하면
구독자에게 알려주는 기능을 사용했습니다.

Kakao SDK

별도의 회원가입 없이 사용자가 손쉽게 서비스를 이용할 수 있도록 카카오 간편 로그인을 사용했습니다.
